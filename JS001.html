Hogyan dolgozik a Böngésző?

A a HTML dokumentum megjelenítése a HTML kód feldolgozásával kezdődik. A Böngésző olvassa a szerverről, vagy a munkaállomás egy könyvtárából a HTML kódot és ez alapján építi a DOM-ot, a HTML dokumentum modeljét. A HTML kódban speciális cimkék / tagek közé zárva JavaScript utasításokat helyezhetünk el. Ez az elem a script elem. Amikor a böngésző egy ilyen elemmel találkozik, értelmezi és végrehajtja azt.

A Js kódban létrehozhatunk egyszerű változókat, vagy összetett objektumokat és a rájuk mutató referenciákat. Ezzekkel a szokásos műveleteket tudjuk elvégezni és a szokásos vezérlő szerkezeteket használhatjuk.

JavaScript módosítja a HTML dokumentumot

Az egyik izgalmas dolg az, hogy a JavaScrittel manipulálhatjuk a felépülésben lévő, vagy márt teljesen felépített DOM-ot. Pl. a HTML kód body elemében elhelyezett Jsel bele tudunk írni a HTML kódba:

Kód

<p><b>Web</b>lap betöltési ideje: <script language="JavaScript" type="text/javascript"> document.write((new Date()).toString()); </script></p>

A document.write() használatára a szövegszerkesztőm panaszokdi. Összeesküdött a Komodo Editemmel a HTML Tidy for HTML5 for Windows version 5.5.31 is, mert minden újraformázákor egy új sort szúr be a script elembe, ami meglehetősen idegesítő tud lenni. A www.w3schools.com azt mondja, ez az utasítás arra való, hogy egy megnyitott ablakba bele írjuk a HTML kódját, így a Böngésző azt megjeleníti. Jó még tesztelésre is, de vigyázni kell, ha akkor adjuk meg, amikor a HTML kód betöltése befejetődött, törli az oldalt. Van más módszer is, hogy a HTML dokumentumot módosítsuk Js-el. Ehhez a HTML és a Js kódnak szorosabban kell együttműködnie. Az a teendő, hogy azt a HTML elemet, amihez valamilyen HTML tartalmat a karunk rendelni el kell látnuk az id azonosító tulajdonsággal. A Js kódban az idnak adott értékkel megkereshetjük a hozzá tartozó DOM objektumot és módosíthatjuk. Nem véletlen a körülményeskedés... nem csak szöveget, hanem HTML kódot is hozzá rendelhetünk pl. egy bekezdés p elemhez. Lássuk a példát:

Kód

<p id="z9_id1"></p> <script> document.getElementById("z9_id1").innerHTML = "<b>Web</b>lap betöltési ideje:"+(new Date()).toString(); </script>

Eredmény

A köny alapján nézzünk példát az idő kezelésére. Az aktuális időpont lekérdezésére a Date osztályt használhatjuk. A new operátorral létrehozunk egy Date típusú példányt és annak toString() metódusával szövegesen megkapjuk a tárolt időt.

v_aktHelyiIdo = new Date();
v_aktHelyiIdoStr = v_aktHelyiIdo.toString();

A toGMTString() az UTC időt adja meg szövegesen.

A document.lastModified tulajdonság a HTML dokumentum utolsó módosításának időpontját... elvileg.

Kód

<p id="z9_id2"></p>
<script>
var v_aktHelyiIdo = new Date();
var v_aktHelyiIdoStr = v_aktHelyiIdo.toString();
var v_utcIdoStr = v_aktHelyiIdo.toGMTString();
var v_utosoModIdoStr = document.lastModified;
document.getElementById("z9_id2").innerHTML = "Utolsó betöltés ideje:"+v_aktHelyiIdoStr+"<br>Utolsó...";
</script>

Eredmény

A var kulcsszó vezeti be változókat. Megadása nem kötelező, de a változók láthatóságának szabályozására használható. Ha egy változót egy függvényen kívül létrehozunk, akkor az a változó írható, olvasható az egész HTML dokumentum (mondhatnánk alkalmazás ?!) működése idején. Ha ugyan ilyen nevű változót szeretnénk használni egy függvényen belül, lokális változóként, akkor a var kulcszóval kell bevezetni, ellenkező esetben a Js értelmező a függvényen kívül bevezetett, globális változót fogja használni annak minden mellékhatásával együtt.

Figyelemreméltó, hogy ha nem Kliens-Szerver konfigurációban vagyunk, akkor az Internet Explorer a HTML dokumentum utolsó módosításának időpontját adja, ellenben az AVAST SafeZone, a Google Crome az oldal utolsó betöltésének / frissítésének időpontját. Kliens-Szerver konfigurációban a szerveren lévő HTML dokumentum utolsó módosításának időpontját kellene kapnunk, de ez függ attól, hogy a szerver helyesen van-e beállítva. Tehát jól meg kell gondolni mire használjuk a document.lastModified tulajdonságot. A Date osztályból származtatott példány viszont a létrejöttének időpontját tárolja. A különbség jó látható, ha ezt az oldalt az Internet Explorerrel futtatjuk.

Nem kötelező a Js kódnak a HTML kódban szerepelnie. A script elem src tulajdonsávával megadhatjuk a külső szkript elérését. A szkript állomány kiterjesztése szokás szerint .js és benne a Js kódok nincsennek script elemben, hiszen ez az elem a HTML szabvány része nem a Jsé.

Kód

<p>Weblap betöltési ideje: <script src="JS001.js"></script> </p>

Szétválasztás

Ahogy nő a projekt, úgy egyre több problémával találkozun. Ezek elkerülésére, vagy legalább könyebb felderítésére ajánlások vannak. Az egyik a var változó meghatározás használata. Egy másik ajánlás, hogy a projekt különböző logikai egységeit külön állományokba tegyük: a HTML kódban a tartalom szerepeljen, ne (vagy csak minimálisan) legyenek formázó vagy aktív elemek. A formázó utasítások a CSS állományokban legyenek. Az szkriptek a Js állományokban.

JavaScript és az Események

A következő izgalmas dolog, hogy különféle eseményekhez hozzárendelhetünk Js kódokat. Pl. egy nyomógomb lenyomása eseményhez hozzárendelhetjük egy figyelmeztető üzenet megjelenítését.

Egy nyomógomb (button) onclick (klikkelés) eseményére egy figyelmeztetést teszünk. A figyelmeztetés egy függvény ami egy kis ablakban megjeleníti a figyelmeztető üzenetet. Visszatérési értéke nincs.

Kód

<button onclicl="window.alert('Huh, jól megklikkeltél :-o');">Klikkelj!</button>

Eredmény

A Klikkelgető gomb működése megmutatja a globális változók használatát. A head elemben elhelyezett script elem tartalmazza a v_klikkSzam globális változót, ami tárolja, hogy hányszor klikkeltünk a gombra. Az onclick esemény megívja a head/script elemben bevezetett klikkeles() függvényt, ami használni tudja a HTML kód betöltésekor létrehozott v_klikkSzam globális változót.

Összefoglalva:

  1. A weboldal törzsébe egy script elembe vagyis tagek közé zárva. A script kimenete / az oldal HTML kódjába kerül. Az így módosított HTML oldal fog megjelenni.
  2. Az oldal fejlécébe (head blokk). Ez esetben a Js kimenet nem jelenik meg. Viszont függvény definíciók, változó deklarációk elhelyezésére nagyszerű.
  3. HTML elemekbe, eseménykezelőkbe (body, form..., stb.). Ilyen esetekben a kódot nem zárjuk scrip elembe.
  4. Önálló állományba. A szokásos állománytípus (kiterjesztés :-)) a .js. Ezt a megoldást a böngészők azzal is támogatják, hogy Js kód a gyorsító tárba (cache) kerül, így a többszöri használata, vagy több lap által való használata gyors lesz.

A HTML kódban elhelyezett JavaScriptek milyen sorrendben futnak?

  1. A head elemben elhelyezett kódok és függvények.
  2. A body elemben elhelyezett kódok a megjelenés(sic!) sorrendjében. Talán helyesebb lenne a sorrakerülésük vagy előfordulásuk sorrendjét írni ?!
  3. Az eseménykezelők csak a hozzájuk rendet esemény bekövetkeztekor indítják el a Js kódot.

Sic, (latinul, jelentése: így) a sic erat scriptum (ez vala írva) kifejezésből. Akkor használják amikor az olvasót arról akarják tájékoztatni, hogy a hibás vagy szokatlan írásmód egy idézett szövegben nem az idézőtől származik, hanem így szerepel az eredeti szövegben.
