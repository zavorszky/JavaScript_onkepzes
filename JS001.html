Hogyan dolgozik a Böngésző?

A HTML dokumentum megjelenítése a HTML kód feldolgozásával kezdődik. A Böngésző olvassa a szerverről, vagy a munkaállomás egy könyvtárából a HTML kódot és ez alapján építi a DOM-ot, a HTML dokumentum modeljét. A HTML kódban speciális cimkék / tagek közé zárva JavaScript utasításokat helyezhetünk el. Ez az elem a script elem. Amikor a böngésző egy ilyen elemmel találkozik, abbahagyja a HTML kód feldolgozását és értelmezi, majd végrehajtja a JavaScript utasításokat.

A JavaScript kód más módon is bekerülhet a HTML kódba. Bizonyos HTML elemekhez eseménykezelőket adhatunk meg úgy, hogy az elem tulajdonságaként megadjuk az esemény nevét (pl.: onload), majd ennek a tulajdonságnak értékül adjuk az esemény bekövetkezését kezelő JavaScript kódot idézőjelek, vagy aposztrófok között.

Az események nevének felépítése:
"on", eseményre_utaló_szó

eseményre_utaló_szó = "click" | "contextmenu" | ... | "touchstart"
(ld.: Események összefoglalója)

Pl. annak az eseménynek a neve, hogy a HTML oldal betöltődött onload. Ehhez az eseményhez hozzákapcsolhatunk egy esemény kezelő JavaScript kódot ami kiírja: A betöltés kész!.
Ilyenkor nem használjuk a script elemet.

JavaScripttel közvetlenül a DOM-ba is elhelyezhetünk eseménykezelőket, ezek a HTML kódban nem jelennek meg, de működnek.

Mivel dolgozik a Böngésző?

A JavaScript kódban létrehozhatunk egyszerű változókat (szám, szöveg tárolására), függvényeket, JavaScript objektumokat (tulajdonságokkal, függvényekkel) és az objektumokra mutató referenciákat. Létrehozhatunk és használhatunk saját objektumokat is (tulajdonságokkal, függvényekkel).

Talán az a legfontosabb, hogy a DOM objektumait is használhatjuk: megváltoztathatjuk építhetjük, lebonthatjuk őket.

Hogy jelenítünk meg adatokat?

Ha a JavaScript módosítja a DOM-ot, akkor a Böngésző a böngésző ablakban megjeleníti a megváltozott DOM-nak megfelelő HTML képet. Az egyik egyszerű gyakorlat az, hogy azt a HTML elemet, amihez a megjelenítendő tartalmat a karunk rendelni ellátjuk az id azonosító tulajdonsággal. A JavaScript kódban az idval megkeressük a hozzá tartozó DOM objektumot (az egyik DOM függvénnyel) és egy értékadással módosíttuk annak tartalmát. Nem csak szöveget, hanem HTML kódot is hozzá rendelhetünk pl. egy bekezdés p elemhez.

Pl.: A2 - JavaScript a body elemben másképp

Feladat

Írjunk ki az oldalra: Hello world!, de készítsün függvényt a feladat elvégzésére.

Példa oldal

JS001A2.html

Leírás

Most nézzük az elejéről a HTML kódot (Klikk a linkre, oldal forrásának megjelenítése...:-)

<!doctype html>
Ez az izé nem is HTML cimke, leginkább deklarációnak mondják. Feladat az, hogy a Böngészőnek megmondja, hogy milyen verziójú HTML kód fog következni. Huh, ez egy nagyon ingoványos terület, legyen elég annyi, hogy a HTML5 kódolást a fenti deklaráció jelzi.

html elem lang tulajdonsággal.
Ez az elem tartalmazza (nyitó és záró cimkéje közrefogja) a teljes HTML kódot. A lang tulajdonság segíti a Böngészőt, hogy a megadott nyelv szerint helyesen mutssa meg az oldalt. Pl.: Egy angol szövegben az idézőjelnek így kell kinéznie: Hello world! egy magyarban pedig így: Üdv világ !

A head elem a HTML lap megjelenítéséhez tartalmaz információkat, kódokat, de nem tartalmaz semmi olyan objektumot, ami közvetlenül megjelenik a lapon.
Az első meta elemet a Tidy HTML kód ellenőrző és formázó program rakta a kódba, hirdeti, hogy ezt a csodát amit láthatunk ő ellenőrizte, ő szépítette. Más szerepe nincs.
A második meta elem a charset tulajdonsággal megadja a karakter készletet, amit a HTML állomány használ, amiben a HTML állomány le van mentve. Ez alapvető, hogy az ékezetes karakterek helyesen jelenjenek meg.
A következő a title elem. Az a szöveg amit tartalmaz (közrefog) megjelenik a HTML dokumentum fülén. Szóval mégis megjelenik valami a HTML oldalon ! Ááááá ! Ez nem az oldal, hanem csak a füle, és különben sem lebet HTML kóddal felturbózni a szöveget. Pl. nem használhatjuk a szöveg kiemelésére a b elemet.
A script elem. Ezért az újdonságért vagyunk itt. Amikor a Böngésző ideér, abbahagyja a HTML kód feldolgozását és a script elemben lévő JavaScript utasításokat értelmezi és végrehajtja. A végrehajtás jelen esetnem azt jelenti, hogy létrehozza a JavaScript függvényünket, és ezután már bárhol a HTML kódban használhatjuk, meghívhatjuk.
A body elem már ismerős, egy sorral rövidebb lett a hozzá tartozó script elem tartalma. No nem ezért az egy sorért érdemes függvényt használni... :-).

A HTML lap ugyanúgy néz ki, de van egy kis különbség. A Hello world vastagítva jelenik meg. Ezzel azt szeretném megmutatni, hogy az innerHTML tulajdonságnak teljes értékű HTML kódot adhatunk meg, nem csak szöveget.

Nézzük meg egy kicsit jobban a függvény deklarációját, ami az alábbi sémának fele meg.

"function", függvény_név, "(", [ paraméterek ], ")", "{",
JavaScript_utasítások,
"}"

paraméterek = paraméter, { ",", paraméter }
JavaScript_utasítások = JavaScript_utasítás, { ";", JavaScript_utasítás }

A függvény tipusáról nem esik szó. A paraméterek tipusáról nem esik szó. Hatókör ? Ugyan. Nyilván szó sem lehet a tipushibák előzetes kiszűréséről. Mi is szürné ki a hibákat, hiszen nincs fordító. Elég messze van a JavaScript a Javatól.
Ez egy másik világ, de meg lehet barátkozni vele.

Kezdjük a barátkozást azzal, hogy megtanuljuk hogyan kell össze konkatenálni egy karakter literált és egy karakteres változót.
(A tipusokról majd később részletesen lesz szó, most maradjunk a példánál.) Az összeragasztást a + operátor végzi: két karakteres kifejezés összege karakteres kifejezés, mégpedig az elsőhöz, hozzáfűzve a második.
Tidy kedves kicserélte a </b> záró cimkémet <\/b> záró cimkére. Bevallom nem értem. A backslash (\) escape (kivétel) karakterként működik a JavaScriptben. pl. az egyszeres idézőjelet a \' karakterpárral kódolhatjuk. Az általam használt böngészők megbirkóznak az én változatommal is, de Tidy egyenlőre okosabb, hiszek neki.

Pl.: A3 - JavaScript eseménykezelő

Feladat

Tegyünk gombot a HTML oldalra. Rendeljünk a gomb klikkelés eseményéhhez (onclick) egy JavaScript programot, ami kiírja a pontos időt.

Példa oldal

JS001A3.html

Leírás

(Klikk a linkre, oldal forrásának megjelenítése...:-)
A JS001A3 megnyitásakor három gobot jelenít meg a böngésző. Hézzük a forrást.
<!DOCTYPE html> - HTML5 a lap.
<html> - A HTML kód nyitó cimkéje az oldal nyelvének meghatározásával.
<html> / <head> - Információk, kódok gyüjtő eleme.
<html> / <head> / <meta> - Tidy... :-)
<html> / <head> / <meta> - A HTML kód karakter kódolása.
<html> / <head> / <title> - A megjelenő lap fülcímkéje.
<html> / <head> / <script> - JavaScript kód

A script elem szükséges, mert a HTML kódot feldolgozó böngészőnek ezzel mondjuk, meg, hogy JavaScript program következik.
A JavaScript program egyetlen függény definícióból áll, a kiir_pontos_ido2-éből. A függvénynek kincs paramétere. Az első sorában létrehozunk egy Date tipusú JavaScript objektumot a new operátorral. Erre az objektumra mutató referenciát értékül adjuk a v_pontos_ido változónak. Nem adunk meg paraméter az objektum létrehozásakor, ezért az aktuális dátum és időpont lesz az objektum (lényegi) tartalma.
A var kulcsszóval jelezzük a JavaScript feldolgozónak, hogy a v_pontos_ido változó a függvényen belül érvényes változó, helyi változó. Igen, igen... mégis van egy kis lehetőség a változók érvényességi körének meghatározására, de ez nem a klasszikus érvényességi kör meghatározás. Hogy milyen lehetőségeink vannak, azt ezzel a példával megtapasztalhatjuk.
A második utasítás a DOM window objektum alert függvényének a hívása. Ez a függvény az adott böngészőlapon megjeleníti a paraméterként megadott üzenetet és egy jóváhagyó klikkelésre vár. A paramétere most egy szöveg literál: A pontos idő (2) : és a dátum objektum toString() függvény eredményének összefűzése.

<html> / <head> / </script> - A script elem lezáró cimkéje.
<html> / </head> - A head elem lezáró cimkéje.
<html> / <body> - A HTML oldal fő vagy törzs része, ebbe kerülnek a megjelenésre szánt HTML elemek.
<html> / <body> / <h1>, </h1> - 1-es szintű címsor.
<html> / <body> / <p> - Egy bekezdés elem nyitó cimkéje. Ez a bekezdés három nyomógomb elemet tartalmaz.
<html> / <body> / <p> / <button>, </button> - Az első button elem a bekezdésben.

Az elem nyitó- és zárócimkéje között van a gomb felirata: Pontos idő(1). A gomb onclick tulajdonságának értékül adunk két JavaScript utasítást macskakörmök között, pontosvesszővel elválasztva. A kód majdnem ugyan az mint a kiir_pontos_ido2 függvény törzsében csak... csak nincs var. Nincs var kulcsszó és nincs függvény sem amiben a v_pontos_ido változó belső változó lehetne. Az így létrehozott változók általános, globális (?) változók, melyek a létrehozásuk HTML lapján használhatók az első értékadástól kezdve.
A Böngésző az eseménytulajdonság értékéül JavaScript kódot vár, ezért a kódon kívül nem kell semmi mást megadni.

<html> / <body> / <p> / ..., <br>, <button>, </button> - A második button elem a bekezdésben.

A gomb onclick tulajdonságában meghívjuk a kiir_pontos_ido2 JavaScript függvényünket.

<html> / <body> / <p> / ..., <br>, <button>, </button> - A harmadik button elem a bekezdésben.

Az utolsó gomb onclick tulajdonságában egy JavaScript utasítás van, mégpedig a DOM window.alert() függvényének megívása. A hívás paramétere a A pontos idő (3) : szöveg literál és a v_pontos_ido változóhoz tartozó dátum objektum toString() függvény értékének egybefűzése. Hoppá! hiszen nem is adunk értéket a változónak! Valóban ennek a gombnak a működése attól függ, hogy előtte melyik gombra klikkeltünk.

<html> / <body> / </p>
<html> / </body>
</html>

Végül szabályosan lezártuk a nyitott HTML elemeket.

Kezdjük a JS001A3 oldal használatát azzal, hogy az (1)-es gombra klikkelün. A Böngésző feldob egy alert (figyelmeztető) ablakot, amiben kiírja a pontos dátumot és időpontot, majd várakozni mkezd, hogy az OK gombbal nyugtázzuk. Jegyezzük meg a másodperceket..., OK..., és kis idő mulva klikkeljünk újra az (1)-es gombra. Igen látszik, hogy múlt az idő, a program szépen teszi a dolgát. Ez volt az az eset, amikor a gomb onclick tulajdonságának értékül adtuk a JavaScript utasításokat.

Folytassuk a klikkelgetést, de most a (2)-es gombra klikkeljünk. Az eredmény ugyan az, mindíg az aktuális időpont jelenik meg az alert ablakban. Ez volt a saját függvény hívás esete.

Ha a színpadon egy pisztoly van, annak előbb-utóbb el kell sülnie. (Molnár Ferenc). Klikkeljünk a (3)-as gombra. Mintha késne az óra... Klikk a (2)-esre, igen késik. Klikk a (3)-asra, hoppá, nem változott az időpont az előbbi (3)-asra klikk óta. Hát persze, hiszen nem adtunk értéket a v_pontos_ido változónak. Akkor most miféle időt ír ez ki? Klikk az (1)-esre és gyorsan OK, majd klikk a (3)-asra. Ez ugyan az ! (Ha elég gyorsak voltunk.) Igen, mert az (1)-es gomb JavaScript kódjában az globális v_pontos_ido változó kap értéket, és mivel globális ezért a (3)-as gomb JavaScript kódjában használhatjuk.

Azért van itt még valami. A JS001 HTML dokumentum kivételével csukjuk be a böngésző füleket és nyissuk meg a JS001A3 lapot a három gombjával. Most kezdjük a klikkelést a (3)-as gombbal. Az általam használt három böngésző egyike sem jelenít meg alert ablakot. Hát persze, mert nincs még v_pontos_ido változó. De a szöveget sem jeleníti meg. Ez azt jelenti, hogy a (3)-as gomb esetén a szöveg konkatenáció nem sikerült, hiba történt és a Böngésző lenyelte a hibát. A window.alert() függvényt már nem hajtotta végre. Jó ez ? Nem jó ? Ilyen a JavaScript.

Szétválasztás

Ahogy nő a projekt, úgy egyre több problémával találkozun. Ezek elkerülésére, vagy legalább könyebb felderítésére ajánlások vannak. Az egyik ajánlás az, hogy a projekt különböző logikai egységeit külön állományokba tegyük: a HTML kódban a tartalom szerepeljen, ne (vagy csak minimálisan) legyenek formázó vagy aktív elemek. A formázó utasítások a CSS állományokban legyenek. A programok JavaScript állományokban.

Pl.: A4 - A HTML-től elválasztott JavaScript

Feladat

Tegyünk gombot a HTML oldalra. Rendeljünk a gomb klikkelés eseményéhhez (onclick) egy JavaScript programot, ami kiírja a pontos időt. A JavaScript kód külön állományban legyen.

Példa oldal

JS001A4.html

Leírás

Bla bla bla...

Összefoglalva:

  1. A weboldal törzsébe egy script elembe vagyis tagek közé zárva. A script kimenete / az oldal HTML kódjába kerül. Az így módosított HTML oldal fog megjelenni.
  2. Az oldal fejlécébe (head blokk). Ez esetben a Js kimenet nem jelenik meg. Viszont függvény definíciók, változó deklarációk elhelyezésére nagyszerű.
  3. HTML elemekbe, eseménykezelőkbe (body, form..., stb.). Ilyen esetekben a kódot nem zárjuk scrip elembe.
  4. Önálló állományba. A szokásos állománytípus (kiterjesztés :-)) a .js. Ezt a megoldást a böngészők azzal is támogatják, hogy Js kód a gyorsító tárba (cache) kerül, így a többszöri használata, vagy több lap által való használata gyors lesz.

Pl.: A5 - Hogyan dolgozza fel a HTLM kódot a Böngésző?

Feladat

Állítsuk meg a HTML oldal feldolgozását betöltéskor.

Példa oldal

JS001A5.html

Leírás

# Avast SZ Chrome IE
1. Üres oldal. Üres oldal. Az oldal első fele megjelenik.
2. Alert ablak az oldal első felénél.
3. Klikk OK.
4. Az oldal első fele megjelenik. Az oldal első fele megjelenik. Az oldal második fele is megjelenik.
5. Alert ablak az oldal teljes betöltéséről.
6. Klikk OK.
7. Az oldal második fele is megjelenik. Az oldal második fele is megjelenik. x

A HTML kódban elhelyezett JavaScriptek milyen sorrendben futnak?

  1. A head elemben elhelyezett kódok és függvények.
  2. A body elemben elhelyezett kódok a megjelenés(sic!) sorrendjében. Talán helyesebb lenne a sorrakerülésük vagy előfordulásuk sorrendjét írni ?!
  3. Az eseménykezelők csak a hozzájuk rendet esemény bekövetkeztekor indítják el a Js kódot.

Sic, (latinul, jelentése: így) a sic erat scriptum (ez vala írva) kifejezésből. Akkor használják amikor az olvasót arról akarják tájékoztatni, hogy a hibás vagy szokatlan írásmód egy idézett szövegben nem az idézőtől származik, hanem így szerepel az eredeti szövegben.
